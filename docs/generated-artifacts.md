# Generated Artifact Policy

Direct Democracy uses generated JSON artifacts for public civic runtime data, admin audits, and local operator evidence. Not every generated file belongs in git.

## Commit These

Commit generated artifacts when they are canonical runtime inputs or stable audit summaries required for public/admin pages:

- `data/generated/*-runtime.json`
- `data/generated/nevada-community-*.json`
- `data/generated/voting-cards.json`
- `data/generated/public-meeting-*.json` when the file is a compact manifest, index, or extracted text artifact used by scripts/pages
- `data/generated/*-audit.json` when it is the latest canonical audit summary without environment suffixes
- `data/generated/*-report.json` when it is a compact deterministic report
- `data/generated/current-officials*.json` when promoted or regenerated by the officials pipeline
- Small raw text/HTML source snapshots when they are intentionally used for reproducible offline imports, such as `data/raw/public-meetings/archive-indexes/*.html` and reviewed official-directory HTML snapshots

Generated public artifacts must not contain secrets, raw identity evidence, raw session cookies, passwords, tokens, or private voter-file rows.

## Keep Local Or Private

Do not commit local/operator-only outputs:

- `.DS_Store`
- `.next/`, `.vercel/`, `node_modules/`, `.local/`
- `*.tsbuildinfo`
- `data/private/**`
- `data/manual-sources/**`
- `data/generated/audits/**`
- `data/generated/admin-operations/**`
- `data/generated/*.codex-sandbox.json`
- `data/generated/*.local-network-enabled.json`
- `data/generated/*.unknown.json`
- duplicate Finder/browser downloads such as `data/generated/* 2.json`
- raw downloaded PDFs, large binary source caches, and provider session artifacts

The meeting cache binary export remains local/private until Vercel Blob is provisioned and the Blob export is intentionally run.

## Review Before Commit

Before committing generated data:

1. Run the relevant audit command for the artifact family.
2. Confirm no secret values are present.
3. Confirm the artifact is either public runtime data, a canonical audit/report, or a source manifest/hash index.
4. Leave environment-scoped run copies untracked.

Useful checks:

```bash
git status --short
git diff --stat
npm run typecheck
npm run build
```
